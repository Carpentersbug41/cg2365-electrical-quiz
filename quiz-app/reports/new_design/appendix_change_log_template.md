# Change Log Template Appendix

Use this template for any shipped prompt, UI, runtime, telemetry, or experiment change.

## Change Record
- `change_id`
- `date`
- `owner`
- `change_type`
- `runtime_version`
- `variant_id`
- `related_report_id`
- `related_experiment_id`

## Reason
- observed problem
- evidence source
- expected effect

## Change Summary
- what was changed
- what was not changed
- risk level

## Evaluation Plan
- primary metric
- secondary metrics
- earliest review date
- rollback condition

## Outcome
- improved metrics
- worsened metrics
- decision: keep / refine / rollback
- notes
